# Elden Bingo Archive

A match archive for Elden Ring bingo games. Store completed boards, track players, and browse match history.

## Features

- Upload and store bingo board screenshots
- Track players and match outcomes
- Add comments to matches
- Browse gallery of past boards
- No account required

## Tech Stack

- React 18 + TypeScript + Vite
- Tailwind CSS
- Framer Motion
- Supabase (PostgreSQL + Storage)
- React Router v6

## Getting Started

### Prerequisites

- Node.js 18+
- npm or yarn
- Supabase account (free tier works)

### 1. Clone and Install

```bash
git clone https://github.com/yourusername/eldenbingo.git
cd eldenbingo
npm install
```

### 2. Set Up Supabase

1. Create a new project at [supabase.com](https://supabase.com)
2. Go to SQL Editor and run the schema from `supabase/schema.sql`
3. Create a storage bucket called `boards` with public access
4. Copy your project URL and anon key

### 3. Configure Environment

```bash
cp env.example .env.local
```

Edit `.env.local` with your Supabase credentials and password:

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
VITE_SUBMIT_PASSWORD=your-secret-password
```

The `VITE_SUBMIT_PASSWORD` is optional but recommended. When set, users must enter this password before uploading matches. Share this password only with your friends to prevent random people from submitting.

### 4. Run Development Server

```bash
npm run dev
```

The app will be available at `http://localhost:5173`

## Database Schema

### Tables

- **players** — Player info (name, color)
- **boards** — Board images with storage paths
- **matches** — Match records with metadata
- **match_players** — Links players to matches
- **comments** — Comments on matches

See `supabase/schema.sql` for the full schema.
